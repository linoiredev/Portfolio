#### How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?
By actively listening during initial consultations to understand the tasks, goals, and challenges users face. Creating user stories helps ensure the program is designed with the user in mind, prioritizing features that provide real value and help users achieve their goals efficiently.

#### How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?
I have been working on a personal note-taking Electron app, which I coincidentally started around the beginning of this class. It is my first time building an app, and I quickly learned the importance of structured development.
At first, I created a wishlist of features and focused on styling the UI before considering tasks complete. This worked at first but became unsustainable as features grew more complex. 

After learning about user stories in class, I realized the value of planning with the user in mind. This inspired me to create wireframes in Miro before coding. I could design layouts, experiment with colors and formatting, and think through functionality in a visual way. Planning this way allowed me to implement features in a logical order, focusing on functionality first and styling second.

I restarted the project using React+Vite with Electron for its modularity. By following a roadmap of prioritized tasks and combining wireframes with user stories, I was able to make significant progress in a week compared to a month with my original approach. This experience highlights Agile principles such as iterative development, prioritization, and designing around user needs, which I hope to continue using in future projects.


#### What does it mean to be a good team member in software development?
Being a good team member means contributing consistently and working honestly. In team environments, it can be easy to shift blame, but the strength of collaboration comes from having multiple people approaching a problem. Team members bring a variety of skill sets and perspectives, which can provide valuable insight and solutions that improve the shared project. Reliability, clear communication, and a willingness to learn from others are key qualities that make a team effective.
