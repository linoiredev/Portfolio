#### Summarize the project and what problem it was solving.
The Corner Grocer Program is a C++ console application that tracks grocery store purchases by reading an input file of daily items and counting how many times each item appears. It stores this data in a map and creates a backup file (frequency.dat) to preserve the results. The program provides a menu for users to search for specific items, view all item frequencies, or see a histogram of purchases. This project solves the problem of efficiently tracking and visualizing item sales, helping store managers quickly identify popular items and trends.
#### What did you do particularly well?
I implemented a fully object-oriented design using a CornerGrocer class, encapsulating all data and functionality. The program is user-friendly with a menu interface, error handling for file operations, and a visually appealing histogram with aligned bars and random colors for better readability. I also included inline comments throughout the code, improving clarity and maintainability.
#### Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
One area for improvement is input validation and sanitization. Currently, the program assumes the input file is properly formatted, but in real-world use, the file might contain unexpected characters, blank lines, or corrupted entries. 

These improvements would make the program more secure, preventing incorrect data from corrupting the frequency counts and ensuring consistent, reliable output. It also makes the program more user-friendly, as users would receive clear feedback when something goes wrong.
#### Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
Creating the histogram with properly aligned bars and random colors was the most challenging. I overcame this by calculating the maximum item name length and adjusting spacing dynamically, as well as experimenting with ANSI color codes for visual distinction. I relied on C++ documentation, online forums, and Stack Overflow for guidance. Going forward, I plan to add resources like C++ style guides, example projects, and GitHub repositories for reference.
#### What skills from this project will be particularly transferable to other projects or course work?
Since this was my first time learning C++, the project helped me build foundational skills in a new programming language that are highly transferable to other projects and coursework. I learned object-oriented programming and class design from the ground up, developing the ability to encapsulate data and functionality effectively. I gained practical experience with file input/output operations and error handling, which will be useful whenever programs need to manage real-world data safely. Working with STL containers like map taught me efficient ways to store and retrieve data, while designing a user-friendly console interface enhanced my understanding of program usability. Additionally, creating a visual histogram gave me experience in basic data visualization, a skill that can be applied to more complex projects in the future.
#### How did you make this program maintainable, readable, and adaptable?
Even as a first-time C++ learner, I focused on making the program maintainable, readable, and adaptable. I encapsulated all core functionality within the CornerGrocer class, separating data logic from the user interface, which makes it easier to modify or extend in the future. I used clear, descriptive variable and method names, and added inline comments explaining the purpose of each section, helping both myself and others understand the code. The menu and histogram systems were built with flexibility in mind, allowing additional features such as new statistical views or alternative visualizations to be added without major rewrites. These design choices ensure that the program is approachable, easy to maintain, and ready for future expansion.
